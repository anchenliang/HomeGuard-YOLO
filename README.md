# HomeGuard-YOLO - 基于YOLOv8的笔记本电脑安防监控系统

**将你的笔记本电脑变成智能安防摄像头，实时监测出租屋动态，发现人员入侵立即邮件通知！**



## 项目介绍

### 背景
长假将至，独自在外租房的你是否也有这样的痛点：
- 出门旅游或回老家，出租屋长时间无人看管
- 担心陌生人闯入，但又不想花钱购买昂贵的监控设备
- 手头正好有一台带摄像头的笔记本电脑闲置在出租屋

**HomeGuard-YOLO** 就是为了解决这个痛点，利用一台带摄像头的笔记本电脑，搭建一套**24小时智能安防监控系统**。它利用 YOLOv8 实时检测画面中的人员，一旦发现入侵者，立即通过邮件发送带有人形标注的现场照片；同时每12小时自动发送一次环境自检报告，让你随时掌握屋内状况。

### 功能特点
- ✅ **实时人员检测**：基于 YOLOv8 模型，准确识别画面中的人
- ✅ **邮件报警**：检测到人时立即发送报警邮件（带标注图片），60秒内不重复报警
- ✅ **环境自检**：每隔12小时自动发送一次自检报告，包含运行统计和当前画面截图
- ✅ **多邮箱通知**：可同时发送给多个收件人（如自己和家人）
- ✅ **本地日志与图片存档**：所有报警和自检图片保存在 `pic` 目录，日志记录在 `log` 目录
- ✅ **实时画面显示**：运行时可看到摄像头实时画面，按 `q` 键退出
- ✅ **零成本改造**：无需购买新硬件，充分利用现有笔记本电脑


## 工作原理

1. **摄像头捕获**：程序打开笔记本自带摄像头（或外接摄像头），每秒采集一帧图像。
2. **YOLO检测**：将采集到的图像送入 YOLOv8 模型，识别画面中的人。
3. **报警触发**：当检测到人且距离上次报警超过60秒，立即发送邮件，并保存标注图片。
4. **定时自检**：启动后1分钟发送首次自检邮件，之后每隔12小时发送一次，报告系统状态和当前画面。
5. **多线程处理**：使用队列和线程分离图像采集、检测、邮件发送和画面显示，保证系统流畅运行。


## 环境要求

- **操作系统**：Windows / Linux / macOS（需支持摄像头）
- **Python**：3.8 或更高版本
- **依赖库**：
  - `opencv-python`：摄像头操作与图像处理
  - `ultralytics`：YOLOv8 目标检测
  - 其他均为 Python 标准库（`smtplib`、`threading`、`logging` 等）

### 2. 安装依赖
建议使用虚拟环境（可选），然后搭建YOLOv8环境（或者其他版本的YOLO），yolo环境搭建过程这里就不细说了，但这是必需的。

### 3. 获取 QQ 邮箱授权码（用于发送邮件）
本程序使用 QQ 邮箱的 SMTP 服务发送邮件，你需要开启邮箱的 SMTP 功能并获取授权码：

1. 登录 QQ 邮箱，点击左上角「设置」→「帐户」
2. 向下找到 **POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务**
3. 开启 **POP3/SMTP服务** 或 **IMAP/SMTP服务**（推荐开启 IMAP/SMTP）
4. 根据提示发送短信，获取**授权码**（16位字母，例如 `xxxxxxxxxxxxxxxx`）
5. 记录下授权码，后续需要填入代码中

> 注意：授权码不是 QQ 密码，务必妥善保管。

### 4. 修改配置信息
编辑 `main.py` 文件，找到 **配置部分**（约第350行左右），填写你的邮箱信息：

```python
email_config = {
    'smtp_server': 'smtp.qq.com',          # 保持不变
    'smtp_port': 465,                       # 保持不变
    'sender_email': '你的QQ邮箱@qq.com',     # 替换为发件人邮箱
    'sender_password': '你的16位授权码',      # 替换为授权码
    # 多个收件人邮箱（列表形式）
    'receiver_emails': [
        '收件人1@example.com',                # 可填自己的另一个邮箱
        '收件人2@example.com'                 # 可填家人的邮箱
    ]
}
```

- **sender_email**：你的 QQ 邮箱地址
- **sender_password**：上一步获取的授权码
- **receiver_emails**：列表形式，可填多个邮箱地址，程序会同时发送给所有人

### 5. 运行程序
在终端执行：
```bash
python main.py
```

首次运行会自动下载 `yolov8n.pt` 模型文件，请保持网络通畅。

程序启动后，会弹出摄像头实时画面窗口，并显示运行时间、检测统计、下次自检倒计时等信息。

- 按 **`q` 键** 可安全退出程序。
- 所有报警图片和自检图片保存在 `pic` 文件夹。

## 使用说明

### 报警机制
- 当画面中出现人且置信度 ≥ 0.5 时，触发报警
- 同一人持续出现在画面中时，为避免邮件轰炸，设有 **60秒冷却时间**
- 报警邮件包含现场标注图片，可清晰看到人形框

### 自检机制
- 程序启动后 **60秒** 发送首次自检邮件
- 之后每隔 **12小时** 发送一次自检报告
- 自检邮件包含：
  - 系统运行时间、启动时间、下次自检时间
  - 累计检测次数、报警邮件发送次数、失败次数
  - 当前环境截图（如果检测到人，图片上会标注人形）

### 摄像头调整
- 默认使用摄像头索引 `0`（通常是笔记本内置摄像头）
- 如果外接多个摄像头，可修改 `camera_index` 参数（例如改为 `1`、`2`）
- 程序在无法打开摄像头时会自动尝试索引 1~4



## 注意事项

1. **摄像头权限**：确保操作系统已允许 Python 访问摄像头（Windows 可能会弹出权限请求）。
2. **网络要求**：发送邮件需要网络连接，若网络不稳定可能导致邮件发送失败，程序会自动重试（通过队列）。
3. **模型性能**：YOLOv8n 是轻量级模型，普通笔记本 CPU 可流畅运行（约 10-20 FPS），但本程序为降低资源占用，每秒仅检测一帧。
4. **图片存储**：`pic` 目录下的图片会无限累积，建议定期清理。
5. **退出方式**：请始终按 `q` 键退出，否则可能无法释放摄像头资源，下次运行时需重启电脑。



## 常见问题

### Q：运行后没有画面窗口？
A：检查摄像头是否被其他程序占用，或尝试修改 `camera_index` 为其他值（如 1、2）。

### Q：邮件发送失败，提示 "Authentication failed"？
A：确认发件人邮箱和授权码是否正确，授权码不是登录密码。部分邮箱（如网易）可能需要开启“允许客户端登录”选项。

### Q：YOLO 模型下载很慢或失败？
A：可以手动下载 `yolov8n.pt` 并放入项目根目录。下载地址：[https://github.com/ultralytics/assets/releases/download/v0.0.0/yolov8n.pt](https://github.com/ultralytics/assets/releases/download/v0.0.0/yolov8n.pt)

### Q：如何修改报警冷却时间或自检间隔？
A：在 `SecurityMonitor` 类的 `__init__` 方法中调整：
- `self.alert_email_cooldown = 60`（秒）
- `self.check_interval = 12 * 60 * 60`（秒）

### Q：程序运行一段时间后卡死？
A：可能是摄像头缓冲溢出或线程阻塞，可尝试降低采集帧率（修改 `frame_interval` 变量）。如有问题欢迎提交 Issue。
